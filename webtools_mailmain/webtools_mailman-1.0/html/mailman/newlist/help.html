Help with Mailman Choices


This is a brief explanation of what the various choices mean when requesting a new Mailman list.  Detailed explanations of the features of Mailman lists may be found in the Mailman documentation.

Owner address: On creation, this is set to the e-mail address associated with the NetID of the person who logged into the newlist tool, in the format netid@rutgers.edu. This mail alias delivers to whatever e-mail address the person has set in his/her Rutgers Online Directory entry. Owners can be changed, or additional owners added, later, if desired, in the list's management pages.
Listname: Enter the listname, all in lower case, including the underscore characters, but without any "@email" addressing.

Correct: list_name
Incorrect: LIST_NAME
Incorrect: listname
Incorrect: list_name@email.rutgers.edu
What steps are required for subscription:
Email confirmation required A potential subscriber will receive an e-mail message which must be replied to before a subscription is permitted.
Require list administrator approval for subscriptions A list administrator must allow each person to subscribe to this list.
Both confirm and approve Both of the above options will apply.  The list administrator must first allow the requester to subscribe, after which the requester must reply to an e-mail message to complete the subscription request.
Who can view subscription list:
Anyone Both members and non-members (including potential spammers) can see subscribers' e-mail addresses.
List members Any address in the subscription list may see the addresses of the other subscribers.
List admin only Only the administrators of this list can see the subscribers' addresses.
Advertise this list when people ask what lists are on this machine: Normally, anyone on the Internet can submit a request (via e-mail or the web) to see the names of lists which are served on email.rutgers.edu. List owners may want this, to let potential subscribers see which lists have a certain word in their titles. Other list owners may want their lists to be hidden from the rest of the world.
No Requests will not display this list's name.
Yes Requests will display this list's name.
Do you want your list to be moderated: Normally, messages are delivered immediately to a list's subscribers. Moderation allows a list administrator to approve or reject all messages before they go out to subscribers.
No Messages sent to the list name are immediately delivered to the subscribers.
Yes Each message sent to the list name must be either approved or rejected by the list administrator.
Action to take for postings from non-members: This controls whether messages from addresses not in the list's subscriber list will be delivered to the list or not.
Accept Messages are accepted for delivery (subject to whether the list is moderated or not).
Hold Messages are held up until a list administrator releases them for delivery or deletes them.  (If a list is not moderated, as in the previous set of choices, this option cannot be used.)
Reject Messages are returned to the sender's address with an error message.
Discard Messages are deleted without notifying the sender or list administrator.
